GeoSync’s landing page is public. Login is only triggered when a visitor wants to contribute data, using OAuth 2.0 Authorization Code + PKCE.

1
Generate PKCE challenge
A random code_verifier and its SHA-256 hash are created client-side before redirecting.
2
Redirect to Clearly.Hub
The user authenticates on auth.clearly.app — GeoSync never sees the password.
3
Exchange code for tokens
The returned code is exchanged for an access_token used on every API call below.
4
Subscription check → wizard
See the Billing tab — only after a plan is confirmed does the optimize-and-publish wizard appear.

GeoSync sends the user to pick a plan immediately after every Clearly.Hub login — no manual "billing" button needed, and no skipping it even if a plan was already chosen before.

1
Tokens issued → forget any prior plan
Right after the OAuth exchange, any previously stored plan is cleared so the check below can’t short-circuit the prompt.
2
Always redirect to the Billing Component
Every login — first time or the hundredth — goes straight to the hosted Billing Component.
3
Redirect back → straight into the wizard
Clearly.Hub redirects back with the chosen plan; the user lands in Step 1 immediately.
This is intentionally re-prompted on every login, even for returning users with an active plan. Each tier also caps how many optimizations you get per plan confirmation — Starter: 1, Professional: 3, Enterprise: unlimited — tracked client-side and reset whenever a plan is (re)confirmed. Use Manage plan in the session panel to change tiers, or check your remaining usage there any time.
